# 项目文档：Solana Intent Pilot (SIP)

## 1. 项目愿景 (The Hook)

SIP 是一个 **Chrome 扩展程序**，它将浏览器变成了通往 Solana 的智能终端。用户无需寻找 DEX 或借贷协议，只需在任何网页上通过自然语言对话，即可由 AI 自动拆解意图、评估风险并执行复杂的链上组合操作。

------

## 2. 混合架构设计 (Hybrid Logic)

我们将系统分为三层，确保性能与智能的平衡：

- **L1 - 推理层 (Cloud Intelligence):** 使用 OpenAI GPT-4o 或 Claude 3.5。负责将模糊的人类语言（“帮我买点那个带猫头像的币”）解析为结构化的 **JSON 意图 (JSON Intent)**。
- **L2 - 验证层 (Local Muscle - Rust/Wasm):** 利用你的 C++ 功底，将逻辑编写为 Rust 并编译为 Wasm 运行在插件中。负责执行实时的本地风控、代币地址合法性校验、以及高频数据过滤，保护用户隐私。
- **L3 - 执行层 (On-chain Skills):** 调用 **Solana Agent Kit** 和 **Jupiter API**。将 L1 产生的意图转化为真正的 Solana 交易指令。

------

## 4. 核心功能实现细节

### 4.1 意图解析流程 (Cloud)

在插件的 `background.js` 中，向 LLM 发送包含当前页面上下文的 Prompt：

TypeScript

```
// 示例：将自然语言转换为结构化指令
const prompt = `You are a Solana DeFi expert. Current webpage text: "${pageContext}". 
User command: "${userCommand}". 
Return a JSON: { action: "swap" | "lend" | "stake", params: { ... } }`;
```

### 4.2 本地执行引擎 (Rust/Wasm)

这是展现你技术实力的部分。你可以用 Rust 编写一个 **Risk-Scanner**：

- **功能：** 检查 Mint Address 的权限是否放弃、流动性是否锁死。
- **优势：** 不经过后端，直接在插件端利用 RPC 完成，速度极快。

### 4.3 交互层 (Solana Actions/Blinks)

将 AI 的建议以 **Blink 卡片** 的形式展示在插件 UI 中。用户点击“确认”即可唤起钱包签名。

## 5. 开发里程碑 (7 天冲刺计划)

### **第一阶段：骨架搭建 (Day 1-2)**

- 使用 Plasmo 初始化项目，配置 `sidepanel`。
- 集成 `solana-wallet-adapter`，实现插件内连接 Phantom。
- **关键任务：** 确保插件能正确读取当前页面的 URL 和选中的文本。

### **第二阶段：大脑集成 (Day 3-4)**

- 搭建简单的 Node.js 中转服务器（用于隐藏 API Key）或直接在插件中使用环境变量。
- 编写 Prompt，实现“识别代币 -> 返回交易参数”的闭环。
- **核心逻辑：** 如果 AI 识别出代币，自动调用 Jupiter 获取报价。

### **第三阶段：Rust 性能优化 (Day 5)**

- 编写 Rust 逻辑，解析 `getAccountInfo` 里的原始 Buffer 数据（利用你的高并发经验）。
- 将逻辑编译为 `.wasm`，集成进 Plasmo 的构建流。

### **第四阶段：Demo 润色 (Day 6-7)**

- 优化 UI 动效（使用 Framer Motion）。
- 录制演示：展示如何从推特上刷到一个币，到 AI 插件自动分析并弹出买入按钮。

## 6. 针对黑客松的“杀手锏”建议 (Winning Strategy)

1. **隐私保护叙事：** 强调你的混合架构中，敏感的交易模拟和风险扫描是在 **本地 Wasm** 环境完成的，用户的私钥相关信息绝不上传云端。
2. **Solana 原生集成：** 既然你之前关注过 **IOTA** 和 **Solana 跨链**，可以加入一个功能：AI 自动识别资产是否在其它链，并推荐最快的 Bridge 路径。
3. **高性能亮点：** 在文档中注明，你使用了 C++ 开发者的思维，通过对 RPC 请求的并发优化，将交易预览的延迟降低到了 300ms 以内。

**目前最重要的一步：** 你现在的机器上是否安装了 `wasm-pack`？如果没有，这是第一步。此外，考虑到你在广州，如果访问 OpenAI API 有网络波动，建议准备一个 **DeepSeek** 或 **OpenRouter** 的 API Key 作为备选方案。
