# Solana Intent Pilot (SIP) 深度开发规格书

## 1. 核心系统架构图

系统分为三个核心垂直层：

1. **感知层 (UI/Content Script):** 负责抓取网页数据（DOM）和用户自然语言输入。
2. **大脑层 (Hybrid Intelligence):** * **云端推理:** GPT-4o 负责意图识别与逻辑拆解。
   - **本地引擎 (Rust/Wasm):** 负责高频数据计算、风险评估和交易模拟。
3. **执行层 (Blockchain Adapter):** 负责签名分发和链上状态同步。

------

## 2. 详细技术实现方案

### 2.1 意图协议格式 (Intent JSON Schema)

为了让云端 AI 与本地执行层完美配合，我们需要定义一套标准指令集。LLM 必须严格返回如下格式：

JSON

```
{
  "intent": "DEFI_SWAP",
  "confidence": 0.98,
  "payload": {
    "inputToken": "So11111111111111111111111111111111111111112",
    "outputToken": "EPjFWdd5AufqSSqeM2qN1xzybapC8G4wEGGkZwyTDt1v",
    "amount": "1000000000",
    "slippage": 0.5,
    "platform": "Jupiter"
  },
  "risk_check_required": true
}
```

### 2.2 本地 Rust/Wasm 验证模块 (The "Safety Guard")

作为 C++ 开发者，这是你的核心竞争力。我们将利用 Rust 的 `anchor-client` 或 `solana-sdk` 编写验证逻辑，并编译为 Wasm 运行。

**核心代码逻辑 (Rust):**

Rust

```
// 验证代币权限是否存在 Rug 风险
#[wasm_bindgen]
pub fn analyze_mint_risk(account_info: &[u8]) -> String {
    let mint = Mint::unpack(account_info).unwrap();
    // 检查是否有 Freeze Authority (冻结权限)
    if mint.freeze_authority.is_some() {
        return "HIGH_RISK: Freeze Authority Enabled".into();
    }
    "LOW_RISK".into()
}
```

### 2.3 插件通信机制 (Plasmo Framework)

插件内部使用 **Long-lived Connections** 保持侧边栏与 Background 的状态同步：

- **Content Script:** 使用 `IntersectionObserver` 监测页面中的 Solana 地址，发现后实时通过 `chrome.runtime.sendMessage` 推送给侧边栏。
- **Side Panel (React):** 监听消息，自动展示代币的 AI 分析报告。

## 3. 核心 API 集成指南

### 3.1 交易执行 (Jupiter API)

不要手动构建 Swap 交易。使用 Jupiter 的 `/quote` 和 `/swap` 接口：

1. **获取报价:** `https://quote-api.jup.ag/v6/quote?inputMint=...&outputMint=...&amount=...`
2. **获取交易体:** 将报价返回的 JSON 传给 `/swap` 接口，获得 Base64 编码的交易。
3. **签名:** 调用 `window.solana.signTransaction`。

### 3.2 链上索引 (Helius API)

用于获取资产余额和解析复杂的 DeFi 历史。

- 使用 **Digital Asset Standard (DAS)** API 获取用户钱包里的所有 NFT 和代币代称、图标，用于 AI 对话时的上下文。

## 4. 详细开发路线图 (24/7 模式)

### Day 1: 核心链路打通

- **Env:** 安装 `plasmo`, `wasm-pack`, `solana-agent-kit`。
- **Wallet:** 实现插件内调用 Phantom 签名。
- **LLM:** 编写并测试 "System Prompt"，确保它能稳定输出 JSON。

### Day 2-3: 业务逻辑层开发

- **Detection:** 实现 Content Script 在 Twitter/Birdeye 上的代币抓取。
- **Execution:** 接入 Jupiter API，实现“意图 -> 报价 -> 确认交易”的闭环。
- **Side Panel:** 仿照 Solana Blinks 的交互样式，制作交易预览卡片。

### Day 4-5: Rust 增强与安全

- **Wasm:** 将 Rust 编写的合约漏洞扫描逻辑编译并集成到插件后台。
- **Simulation:** 使用 Solana RPC 的 `simulateTransaction` 接口，在用户签名钱展示预估的余额变化（Balance Changes）。

### Day 6-7: 演示与提炼

- **Landing Page:** 使用 Vercel 部署一个精美的展示页。
- **Video:** 录制 3 分钟 Demo。重点展示：**AI 自动识别推特代币 -> 一键闪兑 -> 本地风控拦截。**

------

## 5. 黑客松“夺冠”策略 (Pitch Points)

1. **AI-Native UX:** 强调这不再是“点击按钮”，而是“理解意图”。
2. **Performance via Wasm:** 告诉评委，你的风控扫描是在本地毫秒级完成的，不依赖后端，这体现了对 Solana 高性能特性的极致利用。
3. **Intent-Centric:** 紧跟行业热点（Intents），将 SIP 定位为“Solana 上的 Intent 解析层”。
