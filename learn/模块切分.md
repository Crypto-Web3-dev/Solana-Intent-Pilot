为了确保你的项目在黑客松中不仅具备“创意感”，还具备“技术厚度”，我们需要对每一个核心模块进行架构级的详细设计。

作为一名有 C++ 背景的开发者，你可以利用 **Rust/Wasm** 在插件端实现高性能逻辑，这在大多数只用 JS 的项目中会脱颖而出。

------

## 模块一：感知与捕获层 (Web Context & Content Script)

该模块负责从用户当前浏览的网页中提取“金融线索”。

- **技术细节：**
  - **DOM 扫描器：** 使用 `MutationObserver` 监听 DOM 树的变化。针对 Twitter、Birdeye、Dexscreener 编写特定的选择器（Selectors）来精准定位代币符号（Ticker）和合约地址（CA）。
  - **正则表达式引擎：** 实现一个 Base58 编码的高性能正则过滤，识别形如 `[1-9A-HJ-NP-Za-km-z]{32,44}` 的 Solana 地址。
  - **数据总线：** 使用 `chrome.runtime.sendMessage` 将捕获到的 Context（网页标题、当前选中文本、检测到的 CA）实时推送到侧边栏。

## 模块二：AI 意图解析层 (Intent Intelligence)

这是项目的“大脑”，负责将模糊的指令转化为确定的交易参数。

- **Prompt 架构设计：**

  - **System Message:** 定义为一个“冷酷且精准的 Solana 链上量化交易员”。
  - **Few-shot Examples:** 提供 3-5 组（用户输入 $\rightarrow$ 结构化 JSON）的示例，确保 LLM 稳定输出。
  - **动态上下文注入：** 将模块一捕获到的网页信息作为“当前关注点”注入 Prompt。

- **指令协议 (Instruction Schema)：**

  TypeScript

  ```
  interface SIPIntent {
    action: "SWAP" | "LEND" | "STAKE";
    tokenIn: string; // Mint Address
    tokenOut: string;
    rawAmount: string;
    urgency: number; // 1-5，影响滑点设置
  }
  ```

## 模块三：本地 Rust 执行引擎 (Local Wasm Engine)

**这是你的技术核心堡垒。** 所有的安全校验和数据解析不在 JS 中运行，而是在编译后的 Wasm 中运行。

- **核心逻辑设计：**
  - **Risk-Score 算法：** 在 Rust 中实现一个多维评分模型。
    - 维度 1：**Mint Authority**（权限是否丢弃）。
    - 维度 2：**Liquidity Concentration**（流动性池是否过于集中）。
    - 维度 3：**Account Layout Parsing**（利用 `solana-program` 库解析账户数据原始 Buffer）。
  - **Wasm 桥接：** 使用 `wasm-pack` 将 Rust 导出为 ES 模块，通过 `wasm_bindgen` 暴露 `analyze_transaction(data: Uint8Array)` 接口。
  - **优势：** 相比纯 JS，处理大量链上 Buffer 数据的速度提升 10 倍以上，且逻辑不可篡改，安全性更高。

## 模块四：DeFi 执行与适配层 (On-chain Adapter)

负责与 Solana 生态协议进行真实的“握手”。

- **Jupiter V6 集成：**
  - 使用 `quote-api` 获取最佳路径。
  - 实现 **Priority Fee 动态调整**：根据网络拥堵情况，利用 `getRecentPrioritizationFees` 动态设置 `ComputeBudget` 指令，保证黑客松演示时交易 100% 成功。
- **交易模拟器 (Simulation)：**
  - 在正式请求签名之前，调用 RPC 的 `simulateTransaction`。
  - 解析返回的 `logs` 和 `postTokenBalances`，将“预估资产变化”转化为直观的 UI 反馈（如：$-1.0 \text{ SOL} \rightarrow +150 \text{ USDC}$）。

## 模块五：UI/UX 交互层 (Chrome Side Panel)

采用 2026 年最流行的“意图驱动 UI”。

- **交互逻辑：**
  - **侧边栏 (Side Panel)：** 保持常驻，不因为点击网页而消失，适合长对话。
  - **Blink-Style Cards：** 交易确认区域采用卡片化设计。不再展示复杂的 JSON，而是展示“风险等级：低”、“预期收益”、“点击确认签名”。
  - **技术栈：** React + Tailwind CSS + Framer Motion（实现顺滑的 AI 打字机效果和交易状态切换）。

### 关键开发建议：

1. **并发优化：** 在模块一捕获到 CA 的瞬间，后台可以**预异步**请求 Jupiter 的 Quote，这样当用户点开插件时，价格已经刷出来了，体现极致性能。
2. **错误处理：** 在黑客松中，网络往往不稳定。请务必实现重试逻辑（Exponential Backoff）和清晰的错误提示（如：“RPC 节点拥堵，正在切换备用节点”）。
