## Why

当前项目知识主要散落在 `learn/` 目录下，内容存在重复、语气不统一、层级混杂的问题，不利于后续实现、协作与对外展示。现在需要将这些探索性材料收敛为正式项目文档，形成稳定的阅读入口和维护结构。

## What Changes

- 新建 `docs/` 文档体系，按总览、架构、模块、设计、路线图分类整理内容
- 将 `learn/` 中的项目愿景、技术方案、模块拆分和视觉设计统一为一致口径
- 增加文档索引页，明确阅读顺序和后续维护方式
- 为项目文档定义正式能力说明，支持后续持续演进

## Capabilities

### New Capabilities
- `project-documentation`: Define the official documentation structure, scope, and navigation for SIP project knowledge

### Modified Capabilities

## Impact

- Affected systems: `docs/`, `openspec/changes/organize-project-docs/`
- Affected workflows: product alignment, architecture communication, implementation onboarding, demo preparation
- No runtime code or external API behavior changes
