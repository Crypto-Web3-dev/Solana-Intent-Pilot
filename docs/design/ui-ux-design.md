# SIP 界面设计

## 1. 设计目标

SIP 的界面需要同时满足三件事：

- 在狭窄的 Side Panel 中快速建立信息层级
- 把链上复杂度转化为可理解、可确认的动作卡片
- 让 AI、风险扫描和执行状态形成连续的故事线

## 2. 整体设计方向

当前方案沿用 Solana 生态偏科技感的视觉语言，适合作为黑客松 Demo 起点：

- 主题：深色模式 + 玻璃拟态质感
- 品牌感：Solana Purple 与 Solana Green 形成高识别度对比
- 调性：高密度信息面板 + 明确的状态反馈
- 风格关键词：实时、精密、可信、面向交易

## 3. 视觉系统

### 3.1 颜色

| 角色 | 颜色值 | 用途 |
| --- | --- | --- |
| 主背景 | `#0D0D0D` | 应用背景 |
| 次级背景 | `#1A1A1A` | 卡片、输入框、消息气泡 |
| 品牌主色 | `#9945FF` | CTA、重点状态、品牌元素 |
| 安全/成功 | `#14F195` | 风险通过、成功态、收益提示 |
| 危险/警告 | `#FF4B4B` | 风险阻断、错误、取消 |

### 3.2 字体

- 正文：`Inter`
- 数值与地址：`JetBrains Mono`

### 3.3 圆角与质感

- 卡片圆角：12px 到 16px
- 主按钮：胶囊形态
- 面板建议使用弱描边与轻度毛玻璃

## 4. Side Panel 布局

### 4.1 顶部状态栏

内容：

- SIP Logo 与版本号
- 钱包连接状态
- 当前节点或网络状态

作用：

- 建立品牌识别
- 告知用户当前是否具备执行交易的基础能力

### 4.2 实时感知区

内容：

- 当前页面识别到的 token、地址或热点线索
- 一句话 AI 简评
- 风险评分入口或风险仪表

作用：

- 让用户相信系统“看懂了当前页面”
- 为后续意图解析提供上下文可视化

### 4.3 对话区

内容：

- 用户气泡
- AI 流式回复
- 处理中、扫描中等状态占位

作用：

- 承载意图表达和解析反馈
- 保持类似聊天产品的低门槛体验

### 4.4 动作卡片区

内容：

- 交易路由
- 风险结论
- 模拟结果
- 主 CTA 与状态机

作用：

- 把“建议”转成“可执行动作”
- 将签名流程变得可解释

## 5. 核心组件规范

### 5.1 Risk Indicator

展示建议：

- 使用红到绿的连续视觉提示
- 展示 `Mint Auth`、`Freeze Auth`、`Liquidity` 等关键信号
- 支持展开查看详细日志或解释

### 5.2 Action Card

展示建议：

- 显示输入资产、输出资产、估算数量
- 标注协议来源，例如 `Swap via Jupiter`
- 展示模拟结果与滑点
- 基于风险等级切换按钮状态

### 5.3 Detection Bar

展示建议：

- 作为轻量提醒条或跑马灯
- 点击即可把“分析这个代币”填入输入框

### 5.4 Chat Thread

展示建议：

- AI 与用户消息区分明显
- 处理流式文本与状态消息
- 在等待交易或扫描时提供清晰占位反馈

## 6. 关键交互流

### 6.1 正向交易流

1. 页面识别到 token 或地址
2. 用户发出自然语言命令
3. AI 返回结构化意图并生成解释
4. Wasm 返回风险结论
5. UI 展示 Action Card 与模拟结果
6. 用户确认签名
7. UI 显示执行结果与 Explorer 跳转

### 6.2 风险阻断流

1. Wasm 发现高风险信号
2. Action Card 切换为危险状态
3. AI 给出简要风险解释
4. MVP 中用户只能取消或返回，不开放高风险 override

## 7. 前端实现建议

- 使用 React 组件化拆分 Header、ChatThread、ActionCard、RiskIndicator
- 使用 Tailwind 管理颜色、间距和状态类
- 使用 Framer Motion 做卡片推入、消息显现和状态切换
- 把设计 token 抽到主题文件，避免颜色和尺寸散落在组件内

## 8. Demo 优先级建议

如果时间有限，优先保证以下视觉体验：

- 页面检测后有明显感知反馈
- Action Card 具有足够强的中心视觉
- 风险通过与风险阻断两条状态明显可区分
- 成功态能直接展示“交易完成”的正反馈
